## 首次上线步骤（已完成，留作记录 / 重建用）

## 一次性上线（约 10 分钟）

> 在本项目文件夹里执行命令：
> `cd "/Users/elemer/Inbox/compound 专属问卷/compound-rsvp"`

### 1. 登录 Cloudflare（没有账号先去 dash.cloudflare.com 免费注册）
```bash
npx wrangler login
```
浏览器会打开，点 Authorize 授权。

### 2. 创建线上数据库
```bash
npx wrangler d1 create compound-rsvp-db
```
命令会打印一段 `database_id = "xxxxxxxx-..."`。
**把这个 id 复制，粘贴到 `wrangler.toml` 里**，替换 `REPLACE_WITH_YOUR_DATABASE_ID`。

### 3. 给线上数据库建表 + 种入 12 个时段
```bash
npm run db:remote
```

### 4. 部署网站（前端 + API + 后台）
```bash
npx wrangler pages deploy
```
- 第一次会让你确认/创建项目名，建议填 `compound-rsvp`。
- 完成后会给你一个网址，形如 `https://compound-rsvp.pages.dev` —— **这就是交付给老板的链接**。

### 5. 设置管理后台密码（务必做，否则后台无法访问）
```bash
npx wrangler pages secret put ADMIN_PASSWORD --project-name compound-rsvp
```
回车后输入你想要的密码。（可选：再设 `ADMIN_USER`，默认用户名是 `compound`）

设完密码**再部署一次**让它生效：
```bash
npx wrangler pages deploy
```

### 6. 完成
- 问卷： `https://compound-rsvp.pages.dev`
- 后台： `https://compound-rsvp.pages.dev/admin` （浏览器会弹出账号密码框：用户名 `compound` + 你刚设的密码）
- 后台可实时看名单、各时段满座情况，点「下載 CSV」导出。

## ⚠️ 国内访问（最重要）

`*.pages.dev` 默认域名在中国大陆有时会被限速或打不开。上海现场扫码，**强烈建议绑一个自有域名**到这个 Pages 项目：

1. 把域名加到 Cloudflare（域名在哪买都行，改 NS 到 Cloudflare）。
2. Pages 项目 → Custom domains → 添加你的域名（如 `rsvp.compoundlife.ai`）。
3. 二维码用这个自有域名生成。

> 退一步：如果暂时不绑域名，**务必在活动前用上海的网络（手机流量）实测一次** `pages.dev` 链接能否打开。打不开就必须绑域名，或改用香港服务器方案（我可以再帮你做）。

## Day 0 vol.2（2026-07-11）报名页 — 新增（与 vol.1 完全独立，互不影响）

两版设计稿，同一套后端：

- `public/index-a-editorial.html` —「Editorial Duotone」摄影主导、印刷邀请函质感
- `public/index-b-console.html` —「Data Console / Dossier」结构化台账、聚焦点自动追踪的智能体验

本地预览：`npm run dev`，然后打开
`http://127.0.0.1:8788/index-a-editorial.html` 或 `.../index-b-console.html`。

**新增的后端**（完全独立于 vol.1 的 `bookings`/`slots`/`/api/register`，不会动到已有数据）：
- `functions/_lib/pricing.js` — 唯一价格与选项真相来源：`REFERRAL_CODES`、`TIERS`、`TICKET_CHOICES`（沙龍席/會員席/到現場再決定）、`WORKOUT_PREFS`、`TESTING_SLOTS`、`computeAmount()`。`/api/register-v2` 和 `/api/quote-v2` 都从这里 import，改一处两边同步生效，不会出现前后端价格算法各写一份、慢慢跑偏的问题。
- `POST /api/register-v2` — 提交报名，价格在服务端权威计算（永远不信任前端算出的金额）
- `POST /api/quote-v2` — 根据当前表单状态（推荐码 + 席位选择 + 同行人数）实时估价，供页面显示「预估应付」。`GET /api/tier-check?code=` 仍保留（更早、只按单个码查价的版本，标记为 legacy，两版设计稿目前都已改用 quote-v2）。完整推荐码表任何时候都不会下发到浏览器。
- `GET /api/admin/registrations-v2`、`GET /api/admin/export-v2` — 名单 JSON / CSV 导出，走现有 Basic Auth
- 数据表：`registrations_v2`（见 `schema.sql` / `migrations/0002_registrations_v2.sql` 建表，`migrations/0003_v2_expanded_fields.sql` 追加下面这些字段）

**报名表单字段**（对齐《未来人Day0 报名页内容》文档的 13 项）：姓名、微信号、手机/邮箱（选填，二选一皆可）、公司/职位（选填）、邀请人姓名（选填，纯记录不影响价格）、推荐码（解锁同道/VIP价）、席位（沙龍席/會員席/到現場再決定 —— 「到現場再決定」目前按沙龍席價格計算，現場再由人工升級）、會員席專屬的檢測時段選擇、運動工作坊偏好、飲食忌口、偕同道同行（0–2 人，各自需要姓名+微信，人數 ≥1 時全員票價再打 8 折）、當天最想探討的話題（選填）、目前健康管理現狀/問題（選填）。

**上线前必做的 3 件事：**
1. 打开 `functions/_lib/pricing.js`，把 `REFERRAL_CODES` 换成真实推荐码（现在是占位示例，如 `VIP` `FRIEND`）。
2. 用真实收款/企业微信二维码图片替换 `public/assets/wechat-qr-placeholder.svg`（文件名保持不变，两版设计稿都引用它；也可以直接换成同名的 `.png`/`.jpg` 并把两个 HTML 里的一处 `<img>` 路径改一下）。
3. 选定一版设计后，把它另存/改名为 `public/index.html`（或者保留 vol.1 的 `index.html` 不动，改用不同路径分别分享给两批人 —— 由你决定）。

**部署到生产库**（本地 `db:local` 已建好这张表；生产库还没有）：
```
npm run db:remote   # 幂等，只会新增/补齐 registrations_v2 的列，不会碰 vol.1 数据
npx wrangler pages deploy --project-name compound-rsvp --branch main --commit-dirty=true
```

**收款确认**：这次不接 WeChat Pay 在线支付，走「填资料 → 加企业微信 → 转账/发红包」的人工流程（`微信 - 加微信付费`）。提交后 `payment_status` 是 `pending`（VIP 免费票是 `comped`），收到付款后手动标记：
